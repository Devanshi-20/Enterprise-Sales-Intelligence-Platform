## Source Dataset Reality

After profiling the actual source files, the KPI definitions have been
updated to reflect what is truly available in the data:

| Source File | What it contains | KPIs it enables |
|---|---|---|
| `crm_src/cust_info.csv` | Customer ID, name, gender, marital status | CLV, Churn Rate, Customer count |
| `crm_src/prd_info.csv` | Product key, name, cost, product line, dates | Margin %, Product performance |
| `crm_src/sales_details.csv` | Order number, product, customer, dates, sales, qty, price | Revenue, all sales KPIs |
| `ERP_src/CUST_AZ12.csv` | Customer ID, birth date, gender | Age group analysis |
| `ERP_src/LOC_A101.csv` | Customer ID, country | Revenue by country |
| `ERP_src/PX_CAT_G1V2.csv` | Category ID, category, subcategory | Product category reporting |

**Removed from v1.0** (data not available):
- Refund Impact on Net Revenue — no `refund_amount` column in source
- Sales Rep vs Quota — no `sales_rep_id` column in source

**Added in v2.0:**
- Revenue by Country (enabled by LOC_A101)
- Customer Age Group Analysis (enabled by CUST_AZ12 BDATE)

## KPI Catalogue

---

### 1. Total Revenue

| Field | Detail |
|---|---|
| **Definition** | Gross revenue from all sales transactions in the period |
| **Formula** | `SUM(sls_sales)` |
| **SQL** | `SELECT SUM(sls_sales) FROM gold.fact_sales` |
| **Source tables** | `gold.fact_sales` ← from `crm_sales_details` |
| **Filters** | Exclude rows where sls_sales <= 0 (5 rows flagged in Bronze) |
| **Grain** | Daily, monthly, yearly |
| **Owner** | Priya Sharma (CFO) |
| **Refresh** | Daily |

---

### 2. Average Order Value

| Field | Detail |
|---|---|
| **Definition** | Average revenue per sales transaction |
| **Formula** | `SUM(sls_sales) / COUNT(DISTINCT sls_ord_num)` |
| **SQL** | `SELECT SUM(sls_sales) / NULLIF(COUNT(DISTINCT sls_ord_num), 0) FROM gold.fact_sales` |
| **Source tables** | `gold.fact_sales` |
| **Grain** | Monthly |
| **Owner** | Marcus Obi (Finance Analyst) |
| **Refresh** | Daily |

---

### 3. Gross Margin %

| Field | Detail |
|---|---|
| **Definition** | Profitability as a percentage of revenue. Cost sourced from `prd_cost` in product dimension. |
| **Formula** | `(SUM(sls_sales) - SUM(sls_quantity * prd_cost)) / NULLIF(SUM(sls_sales), 0) * 100` |
| **SQL** | Join `fact_sales` to `dim_product` on `product_key` to get `prd_cost` |
| **Source tables** | `gold.fact_sales`, `gold.dim_product` |
| **Filters** | Exclude products where `is_cost_missing = 1` from margin calculation |
| **Grain** | By product line, monthly |
| **Owner** | Marcus Obi (Finance Analyst) |
| **Refresh** | Daily |
| **Target** | > 35% overall |

---

### 4. Month-over-Month (MoM) Revenue Growth

| Field | Detail |
|---|---|
| **Definition** | Percentage change in revenue vs previous calendar month |
| **Formula** | `(This Month Revenue - Last Month Revenue) / Last Month Revenue * 100` |
| **SQL** | `LAG()` window function over monthly revenue CTE |
| **Source tables** | `gold.vw_kpi_revenue` |
| **Grain** | Monthly |
| **Owner** | Priya Sharma (CFO) |
| **Refresh** | Monthly |
| **Target** | > 5% MoM growth |

---

### 5. Year-to-Date (YTD) Revenue

| Field | Detail |
|---|---|
| **Definition** | Cumulative revenue from Jan 1 to today in the current year |
| **Formula** | `SUM(sls_sales) WHERE order_date >= Jan 1 of current year` |
| **Source tables** | `gold.fact_sales`, `gold.dim_date` |
| **Grain** | Cumulative daily |
| **Owner** | Priya Sharma (CFO) |
| **Refresh** | Daily |

---

### 6. Customer Lifetime Value (CLV)

| Field | Detail |
|---|---|
| **Definition** | Total revenue generated by a customer across all their orders |
| **Formula** | `SUM(sls_sales)` grouped by `customer_key` |
| **SQL** | `SELECT customer_key, SUM(sls_sales) AS clv FROM gold.fact_sales GROUP BY customer_key` |
| **Source tables** | `gold.fact_sales`, `gold.dim_customer` |
| **Filters** | Customers with 2+ orders only |
| **Grain** | Per customer |
| **Owner** | Sarah Chen (Sales Manager) |
| **Refresh** | Monthly |

---

### 7. Customer Churn Rate

| Field | Detail |
|---|---|
| **Definition** | % of customers who have not placed an order in 60+ days (amber) or 90+ days (red) |
| **Formula** | `DATEDIFF(DAY, last_order_date, GETDATE()) > 60` → churn risk flag |
| **Source tables** | `gold.dim_customer`, `gold.fact_sales` |
| **Filters** | Only customers with 2+ historical orders |
| **Grain** | Per customer, rolled up monthly |
| **Owner** | Sarah Chen (Sales Manager) |
| **Refresh** | Weekly |
| **Target** | Monthly churn < 3% |

---

### 8. Product Return Rate

| Field | Detail |
|---|---|
| **Definition** | Percentage of sold units that could not be fulfilled or were problematic. Note: the source data does not have a direct returns column — this KPI will be derived from order anomalies (zero/negative sales values) identified during Bronze profiling. |
| **Formula** | `COUNT(sls_sales <= 0) / COUNT(*) * 100` |
| **Source tables** | `gold.fact_sales`, `gold.dim_product` |
| **Grain** | Per product |
| **Owner** | Sarah Chen (Sales Manager) |
| **Refresh** | Daily |

---

### 9. Revenue by Country

| Field | Detail |
|---|---|
| **Definition** | Total revenue grouped by customer country — enabled by joining customer location from ERP LOC_A101 |
| **Formula** | `SUM(sls_sales) GROUP BY country` |
| **SQL** | Join `fact_sales` → `dim_customer` → `dim_location` on `location_key` |
| **Source tables** | `gold.fact_sales`, `gold.dim_location` ← from `ERP_src/LOC_A101.csv` |
| **Grain** | By country, monthly |
| **Owner** | Priya Sharma (CFO) |
| **Refresh** | Daily |

---

### 10. Revenue by Product Line

| Field | Detail |
|---|---|
| **Definition** | Total revenue broken down by product line (Road/Standard/Mountain/Touring) |
| **Formula** | `SUM(sls_sales) GROUP BY prd_line` |
| **Source tables** | `gold.fact_sales`, `gold.dim_product` |
| **Filters** | Use cleaned `prd_line_clean` from Silver (trailing spaces removed) |
| **Grain** | By product line, monthly |
| **Owner** | Sarah Chen (Sales Manager) |
| **Refresh** | Daily |

---

### 11. Customer Age Group Revenue

| Field | Detail |
|---|---|
| **Definition** | Revenue segmented by customer age group — enabled by birth date from ERP CUST_AZ12 |
| **Formula** | `DATEDIFF(YEAR, BDATE, GETDATE())` → age → group (Under 30 / 30–45 / 46–60 / 60+) |
| **Source tables** | `gold.fact_sales`, `gold.dim_customer` ← includes BDATE from `ERP_src/CUST_AZ12.csv` |
| **Grain** | By age group |
| **Owner** | Marcus Obi (Finance Analyst) |
| **Refresh** | Monthly |

---

### 12. Data Quality Pass Rate

| Field | Detail |
|---|---|
| **Definition** | % of automated Great Expectations tests that passed in the most recent pipeline run |
| **Formula** | `passing_checks / total_checks * 100` |
| **Source tables** | `silver.dq_run_log` |
| **Grain** | Per pipeline run |
| **Owner** | Devanshi (Data Engineer) |
| **Refresh** | Per run (daily) |
| **Target** | 100% critical tests / > 98% overall |

## Business Rules

| Rule | Definition |
|---|---|
| **Valid sale** | `sls_sales > 0 AND sls_quantity > 0` |
| **Invalid sale** | `sls_sales <= 0` — excluded from revenue KPIs (5 rows in source) |
| **Active customer** | Customer with at least 1 valid sale in last 12 months |
| **New customer** | Customer whose first order date falls in the reporting period |
| **Churned customer** | Active customer with no order in 90+ days with 2+ historical orders |
| **Churn risk** | Active customer with no order in 60–89 days |
| **Current product** | `prd_end_dt IS NULL` — latest active version of the product |
| **Date format** | Source dates are YYYYMMDD integers — cast to DATE in Silver |
| **Country standard** | US / USA / United States → `United States`; DE / Germany → `Germany` |
| **Gender standard** | M / Male / M (space) → `Male`; F / Female / F (space) → `Female`; else → `N/A` |
